# Research Questions Master

## A. Questions for Paid Courses
1. Extract only the actionable rules from these materials.
2. Ignore storytelling, motivation, and theory unless they directly affect execution.
3. List all setup definitions with:
   - preconditions
   - entry
   - stop
   - exit
   - invalidation
   - session filter
   - market regime
4. Separate mechanical rules from discretionary judgment.
5. Identify all vague concepts that cannot be coded directly.
6. Extract all performance claims and evaluate their evidence quality.
7. Rank the setups by fit for ES/NQ 1-minute intraday scalping.

## B. Questions for External Strategy Research
1. Which strategies are most compatible with ES/NQ 1-minute scalping?
2. Which strategies have complete rules?
3. Which strategies are vague or mostly discretionary?
4. Which strategies overlap with my current system?
5. Which strategies add genuinely new information?
6. Which strategies appear overfit or unrealistic?
7. Which can be adapted into Pine Script strategy format?

## C. Questions for My Current Setup
1. List every indicator and describe its true role.
2. Which indicators are redundant?
3. Which indicators confirm too late?
4. Which indicators may repaint or rely on hindsight?
5. Which parts are objective already?
6. Which parts are subjective?
7. What is the minimum viable version of my current system?

## D. Questions for System Synthesis
1. Rank the top 5 setups most compatible with my style and tools.
2. Which setup ideas should become candidate systems?
3. What should System V1 include?
4. What should be removed?
5. Which logic can be tested immediately?
6. Which logic needs objective proxies first?
7. What are the biggest overfitting risks?

## E. Questions for Risk and Validation
1. What stop model best fits this system?
2. What target model best fits this system?
3. What daily loss cap best protects my account?
4. What hours produce the edge?
5. What hours should be avoided?
6. What is the likely losing streak?
7. What is the expected drawdown range?
8. What sample size is needed before trusting the setup?

## F. Questions for Automation Planning
1. Which parts can already be coded exactly?
2. Which parts need proxies?
3. Which setup is easiest to semi-automate first?
4. What state machine would best model the setup?
5. What alert architecture would be needed?
6. What fail-safes and kill switches would be required?
